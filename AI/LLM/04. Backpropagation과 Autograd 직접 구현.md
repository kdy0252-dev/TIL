# Backpropagation과 Autograd 직접 구현

Forward Propagation이 입력에서 Loss 방향으로 계산한다면 **Backpropagation**은 Loss에서 각 Parameter 방향으로 미분을 전파한다. 목적은 모든 Weight에 대해 “이 값을 조금 바꾸면 Loss가 얼마나 변하는가?”를 구하는 것이다.

---

## 1. 미분과 Gradient

함수 `f(x)`의 미분은 입력을 아주 조금 바꿨을 때 출력이 변하는 비율이다.

$$
f'(x)=\lim_{h\to0}\frac{f(x+h)-f(x)}{h}
$$

```python
def f(x: float) -> float:
    return x**2

x = 3.0
h = 1e-5
numerical_gradient = (f(x + h) - f(x)) / h
print(numerical_gradient)  # 약 6.0
```

입력이 여러 개면 각 입력에 대한 편미분을 모은 Vector가 Gradient다. Gradient는 함수값이 가장 빠르게 증가하는 방향이므로 Loss를 줄이려면 반대 방향으로 이동한다.

$$
\theta \leftarrow \theta - \eta \nabla_\theta L
$$

`η`는 Learning Rate다. 너무 크면 최소점을 지나 발산하고, 너무 작으면 학습이 느리다.

---

## 2. Chain Rule

신경망은 함수의 합성이다.

$$
x \rightarrow z=f(x) \rightarrow y=g(z) \rightarrow L=h(y)
$$

Chain Rule은 중간 함수의 Local Gradient를 곱해 전체 영향을 계산한다.

$$
\frac{\partial L}{\partial x}
=\frac{\partial L}{\partial y}
\frac{\partial y}{\partial z}
\frac{\partial z}{\partial x}
$$

역전파는 출력에서 시작해 이 곱을 반대 방향으로 누적하는 Reverse-mode Automatic Differentiation이다. 입력은 많고 출력 Loss는 하나인 신경망에 효율적이다.

---

## 3. 곱셈과 덧셈의 Local Gradient

`c = a + b`라면 `dc/da = 1`, `dc/db = 1`이다. 위에서 내려온 Gradient를 그대로 양쪽에 전달한다.

`c = a × b`라면 `dc/da = b`, `dc/db = a`다.

```text
upstream = dL/dc
dL/da = upstream × b
dL/db = upstream × a
```

같은 값이 Graph의 여러 경로에 사용되면 각 경로의 Gradient를 더해야 한다. 그래서 구현에서 `grad =`가 아니라 `grad +=`를 사용한다.

---

## 4. 작은 Scalar Autograd Engine

각 `Value`는 값, Gradient, 자신을 만든 이전 Node와 Local Backward 함수를 보관한다.

```python
class Value:
    def __init__(self, data, children=()):
        self.data = data
        self.grad = 0.0
        self.children = children
        self.backward_local = lambda: None

    def __mul__(self, other):
        result = Value(self.data * other.data, (self, other))

        def backward_local():
            self.grad += other.data * result.grad
            other.grad += self.data * result.grad

        result.backward_local = backward_local
        return result
```

전체 구현은 [scalar_autograd.py](./Resources/scalar_autograd.py)에 있다. 다음 Neuron을 Forward한 뒤 역전파한다.

```python
output = tanh(x1 * w1 + x2 * w2 + bias)
output.backward()
```

`backward()`는 Graph를 Topological Sort한 뒤 역순으로 각 Local Backward 함수를 실행한다. 출력 자체에 대한 미분은 1이므로 `output.grad = 1.0`에서 시작한다.

```bash
python3 'AI/LLM/Resources/scalar_autograd.py'
```

## 5. PyTorch Autograd

PyTorch Tensor에 `requires_grad=True`를 지정하면 연산 Graph가 자동으로 기록된다.

```python
import torch

x = torch.tensor([2.0, -1.0])
weight = torch.tensor([0.5, 1.5], requires_grad=True)
bias = torch.tensor(0.1, requires_grad=True)

output = torch.tanh(x @ weight + bias)
loss = (output - 1.0) ** 2
loss.backward()

print(weight.grad)
print(bias.grad)
```

일반적인 Training Step은 다음 순서다.

```python
optimizer.zero_grad(set_to_none=True)
logits = model(inputs)
loss = loss_function(logits, targets)
loss.backward()
optimizer.step()
```

PyTorch는 Gradient를 기본적으로 누적한다. `zero_grad`를 빼면 이전 Batch의 Gradient와 더해진다. Gradient Accumulation을 의도한 경우가 아니라면 매 Step 초기화해야 한다.

---

## 6. `no_grad`, `detach`, In-place 연산

추론에서는 Graph가 필요 없으므로 기록을 끈다.

```python
with torch.no_grad():
    predictions = model(inputs)
```

`detach()`는 같은 Storage를 공유하지만 현재 Graph에서 분리된 Tensor를 반환한다. Loss 값을 Log로 남길 때는 `loss.item()`이 일반적이다.

Backward에 필요한 중간 값을 In-place로 바꾸면 Version 오류가 나거나 Gradient가 잘못될 수 있다. `x += residual`과 `x = x + residual`은 Autograd 관점에서 다를 수 있으므로 Error가 발생하면 In-place 연산을 먼저 확인한다.

---

## 7. Vanishing과 Exploding Gradient

Chain Rule은 많은 Local Gradient를 곱한다. 값이 계속 1보다 작으면 Gradient가 사라지고, 크면 폭발한다.

Transformer는 다음 장치를 사용한다.

- Residual Connection으로 짧은 Gradient 경로 제공
- Layer Normalization으로 Activation Scale 안정화
- 적절한 Weight Initialization
- Gradient Clipping으로 폭발 제한
- GELU 같은 Activation 사용

```python
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

Clipping은 근본 원인을 숨길 수 있다. Learning Rate, Data, Loss Scale과 특정 Layer의 Gradient Norm도 함께 확인한다.

---

## 8. Mixed Precision과 Gradient Scaling

FP16은 표현할 수 있는 작은 값의 범위가 좁아 Gradient가 0으로 Underflow될 수 있다. Loss를 큰 Scale로 곱해 Backward한 뒤 Optimizer Step 전에 다시 줄이는 방식이 Gradient Scaling이다.

```python
scaler.scale(loss).backward()
scaler.unscale_(optimizer)
torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
scaler.step(optimizer)
scaler.update()
```

Gradient Clipping은 반드시 `unscale_` 이후에 적용해야 실제 Gradient 크기를 기준으로 동작한다.

---

## 9. Gradient 검증

작은 함수에서는 Autograd 결과를 Finite Difference와 비교할 수 있다. Double Precision과 작은 `eps`를 사용하는 `torch.autograd.gradcheck`도 제공된다. Custom Autograd Function이나 복잡한 수식을 구현할 때 유용하다.

Loss가 줄지 않을 때는 먼저 다음을 본다.

- Parameter의 `requires_grad`가 `True`인가?
- Loss에서 Parameter까지 Graph가 끊기지 않았는가?
- Gradient가 `None`, 0, NaN 또는 Inf인가?
- Optimizer에 해당 Parameter가 등록됐는가?
- `zero_grad`와 `step` 순서가 맞는가?

---

## 마무리

Backpropagation은 “학습하는 마법”이 아니라 계산 Graph에 Chain Rule을 역방향으로 적용하는 Algorithm이다. PyTorch가 미분을 자동화해도 Tensor가 Graph에 어떻게 연결되는지 이해해야 Gradient 소실, 누적과 Mixed Precision 문제를 진단할 수 있다.
