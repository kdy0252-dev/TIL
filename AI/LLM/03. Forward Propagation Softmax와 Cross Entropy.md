# Forward Propagation, Softmax와 Cross Entropy

Forward Propagation은 입력을 Layer 순서대로 계산해 예측과 Loss를 만드는 과정이다. LLM에서는 Token ID가 Embedding과 Transformer Block을 지나 Vocabulary 크기의 Logit이 된다.

```text
token ids [B,T]
  -> embedding [B,T,C]
  -> transformer blocks [B,T,C]
  -> lm head [B,T,V]
  -> cross entropy scalar
```

---

## 1. Neuron의 Forward 계산

Neuron은 입력의 가중합에 비선형 Activation을 적용한다.

$$
z = \mathbf{x}\mathbf{w} + b, \quad y = f(z)
$$

```python
import math

x = [2.0, -1.0]
w = [0.5, 1.5]
bias = 0.1
z = sum(value * weight for value, weight in zip(x, w)) + bias
y = math.tanh(z)
print(z, y)
```

Linear 연산만 여러 번 쌓으면 전체도 하나의 Linear 변환과 같다. GELU, ReLU, Tanh 같은 비선형 함수가 있어야 복잡한 Pattern을 표현할 수 있다.

Transformer의 Feed Forward Network는 Token마다 독립적으로 다음 연산을 수행한다.

$$
FFN(x)=GELU(xW_1+b_1)W_2+b_2
$$

---

## 2. Logit은 확률이 아니다

마지막 Linear Layer는 Vocabulary의 각 Token에 대한 점수인 Logit을 만든다.

```python
logits = torch.tensor([2.0, 1.0, -0.5])
```

Logit은 음수일 수 있고 합이 1일 필요도 없다. 값의 상대적인 차이가 중요하다. 다음 Token을 하나 선택하려면 이를 확률 분포로 바꿔야 한다.

---

## 3. Softmax

Softmax는 각 Logit을 지수화한 뒤 전체 합으로 나눈다.

$$
p_i = \frac{e^{z_i}}{\sum_j e^{z_j}}
$$

```python
import math

logits = [2.0, 1.0, -0.5]
exponentials = [math.exp(value) for value in logits]
total = sum(exponentials)
probabilities = [value / total for value in exponentials]
print(probabilities, sum(probabilities))
```

지수 함수 때문에 큰 Logit의 차이가 강조된다. 모든 출력은 0과 1 사이이고 합은 1이다.

### 수치적으로 안정적인 Softmax

`exp(1000)`은 Overflow가 발생한다. 모든 Logit에서 최대값을 빼도 결과는 바뀌지 않는다.

```python
maximum = max(logits)
shifted = [value - maximum for value in logits]
exponentials = [math.exp(value) for value in shifted]
probabilities = [value / sum(exponentials) for value in exponentials]
```

PyTorch의 `softmax`와 `cross_entropy`는 이런 안정화가 내부에 구현돼 있다.

---

## 4. Temperature와 Sampling

생성 시 Logit을 Temperature로 나눈 뒤 Softmax를 적용한다.

$$
p_i = softmax(z_i / \tau)
$$

- `temperature < 1`: 큰 Logit이 더 지배적이어서 보수적인 출력
- `temperature > 1`: 분포가 평평해져 다양한 출력
- `temperature -> 0`: 사실상 Argmax에 가까움

```python
temperature = 0.8
probabilities = torch.softmax(logits / temperature, dim=-1)
next_token = torch.multinomial(probabilities, num_samples=1)
```

Training Loss에는 일반적으로 Temperature를 임의로 적용하지 않는다. Temperature는 주로 추론 분포를 조절하거나 Distillation과 Calibration에 사용한다.

---

## 5. Cross Entropy

정답 Token의 예측 확률이 높을수록 Loss가 작아져야 한다. 정답 Class `y`에 대한 Negative Log Likelihood는 다음과 같다.

$$
L = -\log p_y
$$

정답 확률이 0.9면 Loss는 약 0.105지만 0.01이면 약 4.605다. 틀린 답을 확신할수록 큰 벌점을 준다.

```python
target = 0
loss = -math.log(probabilities[target])
```

`CrossEntropyLoss`는 Raw Logit에 `LogSoftmax`와 NLL Loss를 결합한다. 따라서 먼저 Softmax를 적용하면 안 된다.

```python
logits = model(inputs)                    # raw logits
loss = torch.nn.functional.cross_entropy(logits, targets)

# 잘못된 사용
loss = torch.nn.functional.cross_entropy(
    torch.softmax(logits, dim=-1),
    targets,
)
```

---

## 6. LLM의 Next-token Loss

입력 Sequence를 한 칸 이동한 값이 Target이다.

```text
input : [나는, 오늘, 학교에]
target: [오늘, 학교에, 갔다]
```

```python
input_ids = tokens[:, :-1]   # [B, T]
targets = tokens[:, 1:]      # [B, T]
logits = model(input_ids)    # [B, T, V]

loss = torch.nn.functional.cross_entropy(
    logits.reshape(-1, logits.size(-1)), # [B*T, V]
    targets.reshape(-1),                 # [B*T]
)
```

Padding이 포함된 Batch에서는 `ignore_index`를 사용해 Padding 위치를 Loss에서 제외한다. Causal Language Model은 미래 Token을 보지 못하도록 Attention Mask도 별도로 적용해야 한다.

---

## 7. Gradient가 단순해지는 이유

Softmax와 Cross Entropy를 함께 미분하면 Logit `z_i`에 대한 Gradient가 깔끔해진다.

$$
\frac{\partial L}{\partial z_i} = p_i - \mathbb{1}(i=y)
$$

정답 Class는 `p_y - 1`이므로 Logit을 높이는 방향, 나머지는 `p_i`이므로 낮추는 방향으로 갱신된다. 이 Gradient가 Language Model Head와 Transformer Layer를 거꾸로 통과한다.

---

## 8. Loss와 Perplexity

Language Model에서는 Cross Entropy의 지수인 Perplexity를 자주 사용한다.

$$
PPL = e^{loss}
$$

평균적으로 몇 개의 후보 사이에서 혼란스러워하는지를 나타내는 직관을 주지만 Tokenizer와 Dataset이 다르면 직접 비교하기 어렵다. Validation Loss가 낮아져도 실제 생성 품질, 사실성이나 안전성이 자동으로 보장되는 것은 아니다.

---

## 마무리

Forward Propagation은 현재 Weight로 예측을 만드는 방향이고, Loss는 예측이 정답에서 얼마나 벗어났는지를 하나의 Scalar로 압축한다. 다음 단계인 Backpropagation은 이 Scalar에 각 Weight가 얼마나 기여했는지를 계산한다.
