# Self-Attention과 Transformer 구현

Self-Attention은 Sequence의 각 Token이 다른 Token에서 필요한 정보를 가져오는 연산이다. RNN처럼 순서대로 Hidden State를 전달하지 않고 모든 위치의 관계를 행렬 곱으로 계산한다.

---

## 1. Query, Key, Value

입력 Hidden State `X [B,T,C]`를 세 개의 Linear Projection으로 변환한다.

$$
Q=XW_Q, \quad K=XW_K, \quad V=XW_V
$$

- Query: 현재 Token이 찾는 정보
- Key: 각 Token이 어떤 정보를 가지고 있는지 나타내는 색인
- Value: 실제로 섞을 내용

Query와 Key의 Dot Product가 크면 해당 Value를 많이 가져온다.

---

## 2. Scaled Dot-product Attention

$$
Attention(Q,K,V)=softmax\left(\frac{QK^T}{\sqrt{d_k}}+M\right)V
$$

```python
import math
import torch

query = torch.randn(2, 8, 64)     # [B,T,D]
key = torch.randn(2, 8, 64)
value = torch.randn(2, 8, 64)

scores = query @ key.transpose(-2, -1) / math.sqrt(64) # [B,T,T]
weights = torch.softmax(scores, dim=-1)
output = weights @ value                              # [B,T,D]
```

`sqrt(d_k)`로 나누지 않으면 Dimension이 클수록 Dot Product 분산이 커져 Softmax가 한 위치에 포화된다. 포화 영역에서는 Gradient가 매우 작아질 수 있다.

---

## 3. Causal Mask

Decoder-only LLM이 다음 Token을 학습할 때 현재 위치는 미래 Token을 보면 안 된다. Score Matrix의 위쪽 삼각 영역을 음의 무한대로 가린다.

```python
sequence_length = scores.size(-1)
mask = torch.triu(
    torch.ones(sequence_length, sequence_length, dtype=torch.bool),
    diagonal=1,
)
scores = scores.masked_fill(mask, float("-inf"))
weights = torch.softmax(scores, dim=-1)
```

Softmax에서 `exp(-inf)=0`이므로 미래 위치의 확률이 0이 된다. Mask의 방향이 반대이면 과거를 가리고 미래를 보는 심각한 Data Leakage가 생긴다.

---

## 4. Multi-Head Attention

하나의 큰 Attention 대신 Channel을 여러 Head로 나누면 각 Head가 다른 관계를 학습할 수 있다.

```text
입력 [B,T,C]
 -> QKV [B,T,3C]
 -> 분리 [B,H,T,D]
 -> Attention [B,H,T,D]
 -> 결합 [B,T,C]
```

```python
batch_size, sequence_length, channels = hidden.shape
num_heads = 8
head_dim = channels // num_heads

query, key, value = qkv_projection(hidden).chunk(3, dim=-1)

def split_heads(tensor):
    return tensor.view(
        batch_size, sequence_length, num_heads, head_dim
    ).transpose(1, 2)

query, key, value = map(split_heads, (query, key, value))
attended = torch.nn.functional.scaled_dot_product_attention(
    query, key, value, is_causal=True
)
merged = attended.transpose(1, 2).contiguous().view(
    batch_size, sequence_length, channels
)
```

`C`는 `H`로 나누어떨어져야 한다. PyTorch의 Scaled Dot-product Attention은 환경에 따라 Flash Attention 같은 최적화 Kernel을 선택할 수 있어 직접 Score Matrix를 만드는 구현보다 빠르고 Memory 효율적일 수 있다.

---

## 5. Attention의 시간·Memory 복잡도

`QK^T`는 `[T,T]` Score를 만들므로 Sequence Length에 대해 대략 `O(T²)` 시간과 Memory가 필요하다. Context가 두 배가 되면 Score 원소 수는 네 배가 된다.

긴 Context 모델은 Flash Attention, Sliding Window, Sparse Attention, KV Cache와 같은 최적화를 사용한다. Flash Attention은 수학적 결과를 근사하는 기법이 아니라 Tile 단위로 계산해 느린 HBM 접근과 중간 Materialization을 줄이는 Exact Attention Algorithm이다.

---

## 6. Transformer Block

Decoder-only LLM의 한 Block은 보통 다음 구조를 가진다.

```python
class TransformerBlock(torch.nn.Module):
    def __init__(self, dim, attention, feed_forward):
        super().__init__()
        self.attention_norm = torch.nn.LayerNorm(dim)
        self.attention = attention
        self.feed_forward_norm = torch.nn.LayerNorm(dim)
        self.feed_forward = feed_forward

    def forward(self, hidden):
        hidden = hidden + self.attention(self.attention_norm(hidden))
        hidden = hidden + self.feed_forward(
            self.feed_forward_norm(hidden)
        )
        return hidden
```

이 예시는 Pre-Norm 구조다. LayerNorm을 Sub-layer 앞에 두면 깊은 Network의 Training을 안정화하는 데 유리하다.

### Residual Connection

`x + sublayer(x)`는 원래 정보를 보존하고 Gradient가 직접 흐르는 경로를 제공한다. Dimension이 같아야 더할 수 있다.

### Feed Forward Network

Attention이 Token 사이 정보를 섞는다면 FFN은 각 Token의 Channel을 독립적으로 변환한다. 내부 Dimension을 보통 `4C` 정도로 확장했다 다시 `C`로 줄인다. 현대 LLM은 GELU 외에 SwiGLU를 자주 사용한다.

### Layer Normalization

각 Token의 Feature 축을 정규화한다. Batch 간 통계에 의존하지 않아 Sequence 모델에 적합하다. 최근 모델은 RMSNorm도 널리 사용한다.

---

## 7. Encoder와 Decoder-only Transformer

원래 Transformer는 Encoder와 Decoder로 구성됐다.

- Encoder Self-Attention은 양쪽 Context를 모두 본다.
- Decoder Self-Attention은 Causal Mask를 사용한다.
- Encoder–Decoder Attention은 Decoder Query가 Encoder Key·Value를 본다.

GPT 계열은 Decoder-only 구조로 이전 Token만 보고 다음 Token을 예측한다. BERT 계열은 Encoder 구조로 가려진 Token을 양방향 Context에서 복원한다.

---

## 8. KV Cache

생성 단계에서 매 Token마다 이전 Sequence의 Key와 Value를 다시 계산하면 비효율적이다. 각 Layer의 과거 K·V를 Cache하고 새 Token의 Q·K·V만 계산한다.

```text
prefill: Prompt 전체를 병렬 처리하고 KV Cache 생성
decode: 새 Token 하나씩 처리하며 Cache에 K·V 추가
```

KV Cache는 Batch, Layer, Head, Sequence와 Head Dimension에 비례해 GPU Memory를 사용한다. 긴 Context와 큰 Batch에서 Weight보다 KV Cache가 추론 Capacity의 병목이 될 수 있다.

---

## 9. Attention을 Debug하는 방법

- Q, K, V Shape이 `[B,H,T,D]`인가?
- Softmax 축이 Key Sequence 축인 `dim=-1`인가?
- Causal Mask 방향이 맞는가?
- Padding Mask와 Causal Mask가 함께 적용되는가?
- FP16에서 Mask 값과 Softmax가 NaN을 만들지 않는가?
- Head 결합 전에 `transpose`와 `contiguous`가 올바른가?

작은 Sequence에서 Attention Weight를 출력해 각 Row 합이 1이고 미래 위치가 0인지 확인하면 많은 오류를 찾을 수 있다.

---

## 마무리

Transformer의 핵심은 Query와 Key로 정보의 위치를 선택하고 Value를 가중합하는 Attention, Token별 비선형 변환인 FFN, 그리고 깊은 Network를 안정화하는 Residual과 Normalization의 반복이다.
