# Word Embedding과 Token 표현 학습

신경망은 문자열을 입력으로 받을 수 없다. Text를 Token ID로 바꾸고 각 ID를 연속적인 실수 Vector로 변환해야 한다. 이 Vector가 Embedding이다.

---

## 1. One-hot Vector의 한계

Vocabulary가 `['cat', 'dog', 'car', 'bus']`라면 `cat`을 `[1, 0, 0, 0]`, `dog`를 `[0, 1, 0, 0]`으로 표현할 수 있다. 하지만 모든 서로 다른 Token의 거리가 같아 `cat`과 `dog`가 의미상 가깝다는 정보를 담지 못한다. Vocabulary가 100,000개면 Vector도 100,000차원이 된다.

Embedding은 Token마다 작은 Dense Vector를 학습한다.

```text
cat -> [ 0.21, -0.17,  0.83, ...]
dog -> [ 0.19, -0.12,  0.79, ...]
car -> [-0.44,  0.71, -0.08, ...]
```

비슷한 Context에서 나타나는 Token은 학습 과정에서 비슷한 방향의 Vector를 갖게 된다. Word2Vec의 핵심 가정도 단어의 의미가 주변 단어에 의해 드러난다는 Distributional Hypothesis다.

---

## 2. Embedding Layer는 Lookup Table이다

Vocabulary Size가 `V`, Embedding Dimension이 `C`라면 Embedding Weight는 `[V, C]` Matrix다.

```python
import torch
import torch.nn as nn

embedding = nn.Embedding(num_embeddings=10_000, embedding_dim=256)
token_ids = torch.tensor([[10, 25, 91], [7, 25, 3]])  # [B=2, T=3]
vectors = embedding(token_ids)                         # [2, 3, 256]
```

이는 One-hot Matrix와 Weight의 행렬 곱과 수학적으로 같다.

$$
E(token) = one\_hot(token)W_E
$$

하지만 실제 구현은 거대한 One-hot Vector를 만들지 않고 ID에 해당하는 Row를 직접 가져온다. Backpropagation 때 사용된 Row의 Gradient가 계산되고 Optimizer가 해당 표현을 갱신한다.

---

## 3. Word2Vec의 CBOW와 Skip-gram

CBOW는 주변 단어로 가운데 단어를 예측하고, Skip-gram은 가운데 단어로 주변 단어를 예측한다.

```text
문장: the cat sits on the mat

CBOW: [the, sits] -> cat
Skip-gram: cat -> the, sits
```

간단한 Skip-gram은 다음과 같이 만들 수 있다.

```python
class SkipGram(nn.Module):
    def __init__(self, vocabulary_size: int, embedding_dim: int):
        super().__init__()
        self.input_embedding = nn.Embedding(vocabulary_size, embedding_dim)
        self.output = nn.Linear(embedding_dim, vocabulary_size, bias=False)

    def forward(self, center_ids: torch.Tensor) -> torch.Tensor:
        return self.output(self.input_embedding(center_ids))

model = SkipGram(vocabulary_size=50_000, embedding_dim=256)
logits = model(center_ids)                 # [batch, vocabulary_size]
loss = nn.functional.cross_entropy(logits, context_ids)
```

Vocabulary 전체 Softmax는 비용이 크기 때문에 원래 Word2Vec 계열은 Negative Sampling 같은 근사 기법을 사용한다. 실제 LLM은 다음 Token 예측을 통해 Token Embedding을 Transformer 전체와 함께 End-to-end로 학습한다.

---

## 4. 단어가 아니라 Subword Token을 사용한다

단어 단위 Vocabulary는 활용형과 신조어가 생길 때 크기가 폭증하고 미등록 단어 문제가 생긴다. 현대 LLM은 BPE, WordPiece, Unigram 같은 방식으로 자주 등장하는 문자 조각을 Token으로 만든다.

```text
unbelievable -> un + believ + able
playing      -> play + ing
```

Subword는 모르는 단어도 기존 조각으로 표현할 수 있지만 Token 경계가 인간의 단어 경계와 일치하지 않는다. 공백, 대소문자와 Unicode 정규화도 Token 수에 영향을 준다.

학습 Data와 추론에서 같은 Tokenizer와 Vocabulary를 반드시 사용해야 한다. Token ID가 하나만 어긋나도 전혀 다른 Embedding Row를 조회한다.

---

## 5. Position Embedding

Token Embedding만 사용하면 같은 Token 집합의 순서를 구분할 수 없다. Transformer는 위치 정보를 더한다.

```python
batch_size, sequence_length = token_ids.shape
positions = torch.arange(sequence_length, device=token_ids.device)

token_vectors = token_embedding(token_ids)       # [B, T, C]
position_vectors = position_embedding(positions) # [T, C]
hidden = token_vectors + position_vectors         # [B, T, C]
```

Learned Absolute Position Embedding 외에도 Sinusoidal Encoding, Relative Position Bias, RoPE가 있다. RoPE는 Query와 Key를 위치에 따라 회전시켜 상대적 거리 정보를 Attention Score에 반영한다.

---

## 6. Contextual Embedding

Word2Vec의 `bank`는 문장과 관계없이 같은 Vector다. Transformer의 Hidden State는 주변 Token과 Attention을 통과하므로 Context에 따라 달라진다.

```text
river bank   -> 강둑에 가까운 표현
bank account -> 금융 기관에 가까운 표현
```

입력 Token Embedding은 시작점이고, 각 Transformer Layer가 Context를 섞어 더 풍부한 표현으로 바꾼다. 마지막 Hidden State는 다음 Token을 예측하는 Logit으로 Projection된다.

---

## 7. Weight Tying

입력 Embedding `[V, C]`와 출력 Language Model Head의 Weight `[V, C]`를 공유할 수 있다.

```python
self.token_embedding = nn.Embedding(vocabulary_size, embedding_dim)
self.lm_head = nn.Linear(embedding_dim, vocabulary_size, bias=False)
self.lm_head.weight = self.token_embedding.weight
```

입력에서 Token의 의미를 표현하는 공간과 출력에서 Token을 선택하는 공간을 연결하고 Parameter 수도 줄인다. 두 Layer의 Dimension이 같아야 한다.

---

## 8. Embedding을 해석할 때의 주의점

Cosine Similarity가 높다고 인간이 생각하는 모든 의미가 같다는 뜻은 아니다. 학습 Corpus의 빈도, 편향과 Tokenization이 공간에 반영된다. 특정 Dimension 하나를 “감정 축”처럼 단정하기보다 Vector 전체와 Downstream 동작을 평가해야 한다.

Embedding 품질은 최근접 Token 예시만 보지 않고 다음 Token Loss, Retrieval 성능과 실제 Task Metric으로 검증한다.

---

## 마무리

Embedding은 Text를 신경망이 계산할 수 있는 좌표로 바꾸는 학습 가능한 표다. Transformer는 이 좌표에 위치 정보를 더하고 Attention으로 Context를 반복해서 섞는다.
