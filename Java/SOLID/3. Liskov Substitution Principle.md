# Liskov Substitution Principle

LSP는 하위 타입을 상위 타입 자리에 넣어도 Client가 기대하는 계약이 깨지지 않아야 한다는 원칙이다. 단순히 Method Signature가 같다는 뜻이 아니라 사전 조건, 사후 조건, 불변식과 실패 의미가 호환돼야 한다.

## 계약을 깨는 예

```java
public interface BookingPort {
    Either<BookingError, Booking> findById(long bookingId);
}

public final class CachedBookingAdapter implements BookingPort {

    @Override
    public Either<BookingError, Booking> findById(long bookingId) {
        return cache.get(bookingId)
                    .<Either<BookingError, Booking>>map(Either::right)
                    .orElseGet(() -> Either.right(Booking.empty()));
    }
}
```

계약상 존재하지 않는 예약은 `Left(NotFound)`여야 하는데 빈 Booking을 `Right`로 반환한다. Client는 성공값을 실제 Aggregate로 믿으므로 이 구현은 대체 가능하지 않다.

## 호환되는 구현

```java
public final class CachedBookingAdapter implements BookingPort {

    private final BookingCache cache;
    private final BookingPort origin;

    @Override
    public Either<BookingError, Booking> findById(long bookingId) {
        return cache.get(bookingId)
                    .<Either<BookingError, Booking>>map(Either::right)
                    .orElseGet(() -> origin.findById(bookingId)
                                           .peek(cache::put));
    }
}
```

Cache Miss를 “예약 없음”으로 바꾸지 않고 Origin Adapter의 결과를 그대로 보존한다. Cache 장애를 무시할지 실패로 반환할지도 Port 계약에 맞춰야 한다.

## LSP의 계약 규칙

- 하위 타입은 상위 타입보다 더 강한 사전 조건을 요구하지 않는다.
- 하위 타입은 상위 타입보다 더 약한 사후 조건을 제공하지 않는다.
- 상위 타입의 불변식을 보존한다.
- 상위 계약에 없던 예외·`null`·Side Effect를 몰래 추가하지 않는다.
- 시간, 순서와 멱등성도 Client가 의존하면 계약의 일부다.

## Contract Test

같은 Test를 모든 Adapter 구현에 적용하면 대체 가능성을 검증할 수 있다.

```java
interface BookingPortContract {

    BookingPort bookingPort();

    BookingFixture fixture();

    @Test
    default void returnsNotFoundForUnknownId() {
        Either<BookingError, Booking> result = bookingPort().findById(fixture().unknownId());

        assertThat(result.getLeft()).isInstanceOf(BookingError.NotFound.class);
    }

    @Test
    default void preservesSavedDomainState() {
        Booking expected = fixture().cancelledBooking();

        Booking actual = bookingPort().save(expected)
                                      .flatMap(saved -> bookingPort().findById(saved.getId()))
                                      .getOrElseThrow(error -> new AssertionError(error));

        assertThat(actual).usingRecursiveComparison().isEqualTo(expected);
    }
}
```

JPA Adapter, In-memory Fake와 Cache Decorator가 같은 Contract Test를 통과해야 한다. Test Double이 Production보다 지나치게 관대하면 Service Test가 거짓 신뢰를 준다.

## 상속보다 별도 타입이 나은 경우

공통 Field가 있다는 이유로 상속하지 않는다. Read-only Projection과 Mutable Aggregate, 동기 Port와 비동기 Port처럼 계약이 다르면 별도 Interface로 분리하고 Mapper로 변환한다.

## 기억할 점

LSP는 “상속 문법을 올바르게 쓰는 법”보다 넓다. Interface 구현, Adapter와 Test Double도 모두 대체 가능해야 한다. 성공·실패·부재·멱등성과 Side Effect까지 계약으로 보고 Contract Test로 고정한다.
